당신은 React와 소프트웨어 아키텍처에 정통한 수석 프론트엔드 개발자입니다. 
GitHub Pages에 배포할 수 있는 '펫 배치 최적화 시뮬레이터' SPA(Single Page Application)의 전체 코드를 작성해 주세요.
모든 답변과 코드 주석은 **반드시 한국어**로 작성해 주세요.

[아키텍처 및 코드 작성 원칙]
- 관심사 분리(Separation of Concerns) 원칙을 엄격하게 적용하여 비즈니스 로직, UI 컴포넌트, 정적 데이터, 상태 관리를 철저히 분리된 아키텍처로 설계하세요.
- 각 기능과 모듈은 독립적으로 동작하고 재사용 가능하도록 폴더/파일 단위로 나누세요.
- 모든 함수, 커스텀 훅(Hook), 컴포넌트, 특히 복잡한 최적화 알고리즘에는 해당 코드가 왜 이렇게 작성되었고 어떻게 동작하는지 초보자도 이해할 수 있도록 **매우 상세한 한국어 주석**을 달아주세요.

[기술 스택]
- React (Functional Components, Hooks)
- Tailwind CSS (인라인 스타일 및 반응형 UI 디자인)
- State Management: Context API 또는 기능별 Custom Hook 기반의 상태 분리
- 데이터 보존: 브라우저 LocalStorage

[프로젝트 개요]
사용자가 보유한 펫을 4~5개의 장소에 배치할 때, 게임 규칙에 따라 가장 높은 등급(최대 S등급)과 보상을 받을 수 있는 최적의 펫 조합을 찾아주는 시뮬레이터입니다.

[UI 및 핵심 기능 요구사항]
1. 설정 영역: 
   - 파견할 장소 개수 선택 (4개 또는 5개)
2. 보유 펫 선택 영역: 
   - 전체 펫 목록이 그리드로 표시되며, 클릭 시 '보유/미보유' 상태가 토글됨.
   - 보유 상태는 LocalStorage에 즉시 저장되어 새로고침해도 유지되어야 함.
3. 결과 대시보드:
   - '최적화 실행' 버튼을 누르면 연산이 시작됨.
   - 브라우저 멈춤(Freezing) 방지를 위해 연산은 비동기(Web Worker 활용 권장 또는 setTimeout을 활용한 청크 분할 처리)로 진행되며, '계산 중' 로딩 스피너를 표시함.
   - 연산 완료 시, 장소별 배치된 펫, 적용된 능력치/등급, 최종 달성 등급을 카드 형태로 출력.
   - 시간별(4/8/15/22시간) 총 예상 보상 합계를 표 형태로 출력.

[게임 데이터 및 규칙 JSON 로직]
유지보수와 데이터 확장성을 위해 펫 정보, 장소 정보, 등급 판정 룰은 비즈니스 로직에서 분리하여 JSON 객체 배열 형태로 구성해 주세요.

1. 장소 정보:
- 멜룬 축제 즐기기 (요구 능력: 발랄함)
- 물장난치며 놀기 (요구 능력: 유대감)
- 물놀이 하며 추억 쌓기 (요구 능력: 배려심)
- 파란 장비 화관 만들기 (요구 능력: 예리함)
- 돌 조각하기 (요구 능력: 발랄함, 자신감 중 높은 것 적용)
* 활발함은 발랄함으로 취급.

2. 등급 판정 조건:
- 장소당 최대 3마리 배치 가능 (같은 펫 중복 배치 불가).
- 요구 능력이 없는 펫은 D등급으로 계산.
- S등급 조건: (A×1 + B×2), (A×2 + D(희귀)×1), (A×2 + C×1), (S + B(전설) + C), (S×1 + A×1)
- A등급 조건: (B(고급) + C(희귀) + C), (B(전설1, 고급1) + D×1), (A + C), (A×1 + D(전설1, 희귀1)), (A×1 + B×1), (S×1 + D×2), (S×1 + C×1)
- B등급 조건: (D×3(전설2, 희귀1)), (C×1 + D×2(희귀2 또는 전설1, 일반1)), (C×2(고급1, 일반1)), (B×1 + D×1), A×1, S×1
- C등급 조건: (D×1(전설)), (D×2(일반1, 희귀1)), (D×3(일반2, 고급1)), C×1, B×1
- D등급 조건: D×1(일반~희귀)

3. 시간별 보상 (D / C / B / A / S 순):
- 4시간: 1 / 1~2 / 2 / 2~3 / 3
- 8시간: 2 / 2~4 / 3~4 / 4~5 / 5~6
- 15시간: 3~4 / 5~6 / 6~7 / 8 / 10
- 22시간: 5 / 7~8 / 9 / 11 / 14

[알고리즘 최적화 및 2차 판단 기준]
- 조합 폭발(Combinatorial Explosion)을 막기 위해, 백트래킹(Backtracking) 및 가지치기(Pruning)를 적용하여 해당 장소의 요구 능력치가 아예 없는 펫은 1차적으로 탐색에서 배제하세요.
- 최고 등급 조합이 여러 개일 경우, 아래 우선순위에 따라 결과를 1개만 반환하세요.
  1순위: 전체 장소 등급 점수 합이 가장 높은 것 (S=4, A=3, B=2, C=1, D=0)
  2순위: 펫을 가장 적게 사용한 조합
  3순위: 전설 등급 펫을 덜 사용한 조합

[요청 산출물 구조]
아래와 같이 폴더 구조를 가상하여 각 파일별로 코드를 작성해 주세요.
1. `src/constants/data.js` : 정적 데이터 (펫, 장소, 규칙)
2. `src/utils/optimizer.js` : 최적화 핵심 알고리즘 및 점수 산정 비즈니스 로직
3. `src/hooks/usePets.js` : LocalStorage 연동 및 펫 상태(선택/해제) 관리를 담당하는 커스텀 훅
4. `src/components/` : UI 컴포넌트 분리 (예: `Header.jsx`, `PetGrid.jsx`, `ResultBoard.jsx` 등)
5. `src/App.jsx` : 메인 레이아웃 및 상태 통합
6. GitHub Pages 배포를 위한 간단한 `package.json` 스크립트 및 설정 가이드
